## 2. ארכיטקטורה ומבנה כללי

### 2.1 מבנה UI כללי
- **Layout עיקרי:** תפריט צד ימני קבוע (collapsible) + אזור תוכן מרכזי
- **Header עליון:** 
  - לוגו המערכת (משמאל)
  - שם המוסד + תווית "דמו" (במרכז)
  - כפתור צילום מסמך (מימין)
  - כפתור תפריט המבורגר (קיצוני ימין)
- **תמיכה ב-RTL:** כל הממשק בעברית עם תמיכה מלאה בכיוון ימין לשמאל
- **עיצוב:** Material Design עם ערכת צבעים מקצועית (כחול, ירוק, אדום להתראות)

### 2.2 תפריט ניווט ראשי (Side Menu)

**מבנה התפריט:**
1. **דשבורד** - `/dashboard` - מסך ראשי עם סקירה כללית
2. **רופא/ה** - `/doctorhome` - ממשק רופא לניהול דיירים
3. **מקצועי** - `/professional` - ממשק מקצועי רפואי (פיזיו, ריפוי בעיסוק וכו')
4. **לוח שנה** - `/calendar` - ניהול אירועים וישיבות צוות
5. **דיירים** - `/residents` - רשימת דיירים וניהול מלא
6. **אח/ות** - `/nursehome` - ממשק אחות עם כלים סיעודיים
7. **דיווח סיעודי שוטף** - `/nursingshiftreport` - דיווחים יומיומיים
8. **חלוקת תרופות** - `/consolidatedcardexselector` - מערכת ניהול תרופות
9. **צוות** - `/staff` - ניהול צוות המוסד
10. **כללי** - `/general` - הגדרות כלליות
11. **הגדרות** - `/settings` - הגדרות מערכת
12. **נהלי אבטחת מידע** - `/securityprocedures` - מסמכי אבטחה
13. **פתקים** - `/stickynotes` - מערכת הודעות פנימית
14. **פתקים אליי** - כפתור לסינון פתקים אישיים
15. **ישיבות צוות** - `/multidisciplinarymeetings` - ניהול ישיבות רב-תחומיות

  const interactions = [];
  
  for (let i = 0; i < medications.length; i++) {
    for (let j = i + 1; j < medications.length; j++) {
      const interaction = findInteraction(

**קטגוריות:**
- < 18.5: תת משקל
- 18.5-24.9: תקין
- 25-29.9: עודף משקל
- 30-34.9: השמנה דרגה 1
- 35-39.9: השמנה דרגה 2
- 40+: השמנה מורבידית

**משקל אידיאלי (נוסחת Devine):**
- גברים: 50 + 2.3 × (גובה בסנטימטרים - 152.4) / 2.54

- טיפול נכון בטבלאות ורשימות
- Flexbox/Grid עם `direction: rtl`
- תמיכה בטקסט מעורב (עברית + אנגלית במספרי תרופות)
- Date/Time formatters מותאמים לעברית

**Responsive Design:**
- Mobile-first approach
- Breakpoints:
  - Mobile: 320px - 767px
  - Tablet: 768px - 1023px
  - Desktop: 1024px+
- Touch-friendly buttons (מינימום 44x44px)
- המבורגר menu במובייל

**Performance Optimization:**
- Virtual scrolling לרשימות ארוכות
- Image lazy loading
- Code splitting per route
- Debounce על שדות חיפוש (300ms)
- Memoization של קומפוננטות כבדות
